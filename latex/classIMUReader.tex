\doxysubsection{Detailed Description}
\doxylink{classIMUReader}{IMUReader} provides a threaded, event-\/driven wrapper around the low-\/level ICM-\/20948 driver.

SOLID principles\+: S -\/ This class is responsible for coordinating asynchronous IMU sample acquisition from the data-\/ready GPIO and forwarding decoded samples to client code. O -\/ New behaviours can be attached through the callback interface without changing the reader logic itself. D -\/ The low-\/level register and I2C interaction remain inside the IMU driver, while this class focuses on threading and sample dispatch.

Threaded IMU reader driven by the data-\/ready GPIO.

Background worker loop waiting for DRDY events and dispatching samples. This keeps blocking/event-\/loop behaviour internal to the class so client code does not need to manage low-\/level synchronisation details. epoll waits on both the GPIO event fd and the shutdown event fd.

Single-\/slot buffer is enough since events are handled one at a time.

hold the shared I2C bus mutex for the full read\+\_\+sample() call so the ADS1115 thread cannot interleave its own ioctl(\+I2\+C\+\_\+\+SLAVE) + read between our register writes.

Optional DRDY status check after the sample read.
